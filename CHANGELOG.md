# Changelog

## v0.1.13-rev.4 - 图片转ICO功能 - 2026-03-12

- feat: 编辑页拖拽图片文件时新增图片转 ICO 流程，支持 SVG、PNG、JPG、JPEG、BMP、WebP，并生成包含 16/24/32/48/64/128/256 的单文件 ICO。
- feat: 图片拖拽悬停提示扩展为三种模式卡片：高度顶满、宽度顶满、自定义裁切；前两种松手直接生成并应用，自定义模式进入裁切弹窗。
- feat: 手动裁切接入通用弹窗体系，支持 1:1 裁切框、四角与四边手柄、双击重置以及圆角裁切参数，并将结果直接用于最终 ICO 输出。
- feat: 生成结果默认保存到当前文件夹的 `.ICON` 目录并立即应用，同时在编辑页新增“清空图标缓存（xxxMB）”入口用于清理当前目录图标缓存。
- style: 拖拽提示与图片裁切弹窗的文案、卡片布局和交互层级已统一纳入现有弹窗体系，并补齐图片转图标场景的 i18n。

## v0.1.13-rev.3 - 更好的内存与GPU性能优化 - 2026-03-11

- feat: 新增手动启停的 memory profile 输出器与 profile 输出目录入口，内存排查改为按需记录，不再默认常驻。
- perf: 收口最小化与后台 suspend 链路，移除会导致后台 GPU 异常升高的自定义最小化动画，仅保留稳定的恢复回弹动画。
- perf: 修复置顶按钮呼吸光环缺少帧率约束导致的整窗高刷刷新问题，置顶状态下的 GPU 占用与窗口帧率恢复受控。
- perf: 修复编辑页面静止时仍维持高帧率刷新的问题，滚动结束后可正确回落到低频空闲状态。
- perf: 增加空闲回收触发点并收敛内存排查链路，在控制综合内存占用的同时降低空闲阶段的额外干扰。
- style: 调整主页最近编辑滚动条的对比度与圆头样式，降低默认状态的扎眼程度。

## v0.1.13-rev.2 - 场景化帧率调度优化 - 2026-03-10

- perf: 完成场景化自动帧率调度，前景刷新统一按 `Static / Active / Max` 门控，主页、编辑页与置顶态在静止时都能正确回落，不再长期跑高刷。
- perf: 背景流光、主页标题流光、置顶按钮光环与编辑页管理员标题流光拆分为独立低频通道，并下调背景默认频率，降低高刷新率显示器下的 GPU 合成压力。
- perf: 修复编辑页隐藏 `ProgressBar`、置顶光环过渡和局部弹层动画误带起整窗高频重绘的问题，高频刷新只在滚动、拖拽和真实过渡期间激活。
- perf: 性能采样改为按需启用、后台异步和低频发布，关闭监视器后不再常驻 CPU/GPU 计数器轮询。
- refactor: 首页与编辑页视图改为按需创建/释放，并补充空闲期保守内存回收，减少启动期和页面切换时的常驻资源占用。
- fix: 修复语言菜单的打开/关闭动画冲突、失焦收回与弹窗区域误拦截窗口手势的问题，恢复弹窗状态下的窗口拖动与置顶切换行为。
- style: 主页最近编辑滚动条改为局部样式调整，恢复正确圆角和更柔和的默认可见度，不再影响其他页面滚动条。

## v0.1.13-rev.1 - 拖拽悬停与配置重构 - 2026-03-09

- feat: 重构拖拽悬停反馈，新增统一 `DragIntent` 分析服务并按主页/编辑页上下文区分可接收内容。
- feat: 复用通用弹窗体系实现无按钮拖拽提示态，支持 `DismissOnEsc`、按钮显隐、遮罩点击关闭、中心图标与副文案配置。
- feat: 主页面拖拽只允许单文件夹/多文件夹；多文件夹显示“开发中”提示且本版本不执行 Drop 动作。
- feat: 编辑页拖拽支持单/多文件夹、ico、exe（内/外目录）、文本别名与不支持内容的差异化反馈。
- perf: 拖拽高频路径中的 exe 图标检测改为异步 + 缓存，降低连续 DragOver 卡顿风险。
- refactor: 配置存储迁移至 `%LocalAppData%\\FolderStyleEditorForWindows\\`，实现“新路径优先、旧路径回溯、无配置自动生成”。
- refactor: 配置保存改为文本增量更新，避免整文件序列化导致的注释与格式丢失。
- refactor: 历史记录与别名历史迁移至新数据目录，并兼容旧路径读取后自动写回。
- style: 下调编辑页图标选择器选中悬停高亮强度，修复主题色叠加过亮问题。
- style: 拖拽提示层支持基于鼠标位置的轻微联动位移，提升悬停反馈质感。
- style: Toast 退场动画保持原有缩高效果，并将显示位置微调到不遮挡编辑页小标题的安全区。
- fix: 双击置顶改为“按下记录、移动触发拖动、松手计数”的统一手势状态机，避免拖动窗口被误判为双击点击。
